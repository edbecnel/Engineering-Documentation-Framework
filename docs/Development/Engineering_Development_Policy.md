# Engineering Development Policy

> **Status:** Canonical
> **Owner:** Engineering Documentation Framework
> **Applies To:** All projects adopting EDF

## Purpose
Define the engineering principles used throughout the project lifecycle.

## Principles

1. Repository is the single source of truth.
2. Architecture before implementation.
3. Documentation-first.
4. Incremental improvement.
5. Human approval is required for significant engineering decisions.
6. Validate before release.
7. Record significant changes.
8. Existing project-owned content is never overwritten automatically.

## Documentation-First Principle

Engineering work should normally follow:

1. Architecture
2. Documentation
3. Implementation
4. Validation
5. Change recording
