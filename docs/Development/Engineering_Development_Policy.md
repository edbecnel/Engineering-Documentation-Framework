## Purpose

Define the engineering principles used throughout the project lifecycle.

## Principles

1. The repository is the single source of truth.
2. Architecture precedes implementation.
3. Development is documentation-first.
4. Improvement is incremental.
5. Human approval is required for significant engineering decisions.
6. Changes are validated before release.
7. Significant changes are recorded.
8. Existing project-owned content is never overwritten automatically.

## Documentation-First Principle

Engineering work should normally follow:

1. Architecture
2. Documentation
3. Implementation
4. Validation
5. Change recording
